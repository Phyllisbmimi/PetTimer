## 🧪 完整測試流程

### 測試 1: 認證系統

#### 1.1 註冊新帳戶
1. 打開 http://localhost:5175
2. 點擊「註冊」按鈕
3. 輸入：
   - 郵箱：test@example.com
   - 用戶名：TestUser
   - 密碼：123456
   - 確認密碼：123456
4. 點擊「註冊」
5. **預期結果**：進入郵箱驗證頁面

#### 1.2 郵箱驗證
1. 檢查瀏覽器控制台（Console）
2. 看到驗證碼（6位字母）
3. 輸入驗證碼
4. 點擊「驗證電郵」
5. **預期結果**：自動跳轉到登入頁面

#### 1.3 登入帳戶
1. 輸入：
   - 郵箱：test@example.com
   - 密碼：123456
2. 點擊「登入」
3. **預期結果**：成功登入，進入寵物選擇頁面

#### 1.4 密碼重設（可選）
1. 在登入頁面點擊「忘記密碼？」
2. 輸入郵箱：test@example.com
3. 點擊「發送驗證碼」
4. 在控制台找到重設碼
5. 輸入驗證碼
6. 設置新密碼
7. **預期結果**：成功重設密碼，返回登入頁面

## 🐛 已知問題和限制

### 1. 郵箱驗證
- ⚠️ 目前**沒有真實的郵件發送**
- 驗證碼顯示在**瀏覽器控制台**（開發模式）
- 正式環境需要集成郵件服務（SendGrid, AWS SES 等）

### 2. 密碼安全
- ⚠️ 密碼使用簡單的 base64 編碼（僅示例）
- 正式環境需要使用**後端 API** 和真正的加密算法（bcrypt）

### 3. 數據儲存
- 💾 所有數據存儲在 **localStorage**
- 沒有後端數據庫
- 清除瀏覽器數據會丟失所有信息

### 4. 多用戶
- 👥 目前支持多用戶（通過郵箱區分）
- 但所有數據都在本地，沒有雲端同步

## 📋 開發者 Debug 提示

### 查看 localStorage 數據
```javascript
// 在瀏覽器控制台執行：

// 查看所有 localStorage 鍵
Object.keys(localStorage)
  .filter(key => key.startsWith('petTimer'))
  .forEach(key => console.log(`${key}:`, localStorage.getItem(key)));

// 查看當前登入用戶
console.log('Auth User:', localStorage.getItem('petTimer_auth_user'));

// 查看語言設置
console.log('Language:', localStorage.getItem('petTimer_language'));
```

### 清除所有數據重新開始
```javascript
// 清除所有 PetTimer 數據
Object.keys(localStorage)
  .filter(key => key.startsWith('petTimer'))
  .forEach(key => localStorage.removeItem(key));

// 然後刷新頁面
location.reload();
```

### 快速跳過郵箱驗證
```javascript
// 在註冊後，在控制台執行：
const email = 'test@example.com';
const userData = JSON.parse(localStorage.getItem(`petTimer_user_${email}`));
userData.user.emailVerified = true;
localStorage.setItem(`petTimer_user_${email}`, JSON.stringify(userData));
console.log('Email verified!');
```
